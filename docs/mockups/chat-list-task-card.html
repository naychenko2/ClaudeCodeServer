Карточка чата-задачи в списке

У чата, порождённого задачей (origin = task), заголовок задачи звучит трижды: в названии чата («Задача: …»), в плашке происхождения (🔧 «Задача: …») и косвенно — техническим промптом в превью («## ЗАДАЧА Выполни…»). Карточка раздувается до 4 строк, а стопка таких карточек в иерархии выглядит стеной повторов. Ниже — текущее состояние и 4 варианта редизайна. Все цвета — из токенов lib/design.ts / theme.css (светлая тема).

  • Вариант A — «Одна правда». Заголовок один раз, «Задача» → тихий ключ-признак, превью → живой статус выполнения. 2 строки вместо 4. Рекомендуется.
  • Вариант B — «Квитанция». Однострочная карточка-прогон: чаты-задачи визуально отделены от «живых» разговоров.
  • Вариант C — «Задача-узел». В дереве задача становится группирующей шапкой, чаты-исполнители вложены под ней без повтора заголовка.
  • Вариант D — «Прогресс». Строка статуса как тонкий трек подзадач — надстройка над A, где у задачи есть подзадачи.
Данные на макете — с реального скриншота: оркестратор «Создание и запуск экспертной команды» и три вложенных чата-задачи. Шрифты: PT Serif / Hanken Grotesk / JetBrains Mono. Тёмная тема не показана — цвета берутся из CSS-переменных автоматически.

Вариант A — «Одна правда» рекомендуется

Заголовок задачи — ровно один раз в строке названия, взят из task.title (без префикса «Задача:»). Тип элемента показывает тихий ключ-признак 🔧 без фона и текста (весь заголовок — в тултипе). Строка превью с промптом заменена на живой статус выполнения: выполняется / ждёт ответа / готово / в очереди, плюс прогресс подзадач и срок. Итого 2 строки вместо 4.

Вариант B — «Квитанция»

Чат-задача — это «прогон», а не разговор. Даём ему однострочную карточку, визуально отличную от обычных чатов: статус-точка · ключ · заголовок · справа мини-аватар исполнителя (18px вместо крупного лица) + чип статуса/срока. Максимальная компактность; чаты-задачи явно отделены от «живых» разговоров.

Вариант C — «Задача-узел»

Надстройка над A для дерева: когда под оркестратором несколько исполнителей одной задачи — сама задача становится группирующей шапкой (serif-заголовок, прогресс, chevron), а чаты-исполнители вложены под ней без origin-признака (шапка уже сказала, чья это ветка). Заголовок задачи живёт один раз на всю группу. В плоском виде деградирует к варианту A.

Шапка появляется только при ≥2 прогонах одной задачи; одиночная задача (нижняя) остаётся карточкой A — без лишнего уровня.
Плюсы: самый сильный ответ на «стену повторов» — заголовок один раз на группу; serif-шапка задаёт «документный» ритм.
Минусы: требует группировки узлов по taskId в chatTree.ts — это уже логика дерева, а не косметика. Внедрять поверх A, когда дойдут руки.

Вариант D — «Прогресс»

Декоративная надстройка над строкой статуса A: вместо текста — тонкий трек подзадач (4px) с подписью N/M. Мгновенно читается «насколько готово». Применимо только там, где у задачи есть подзадачи; иначе — фолбэк на текст-статус A.
