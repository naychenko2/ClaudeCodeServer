## Порядок исполнения

**R1 → R2 → R3 → R4** (backend-граница + безопасность; тестируется `module-echo` без фронта) → **R7** (MCP) → **R5a → R5 → R6** (фронт; R5a — первым) → **R8, R9** — сквозные.

## Регрессионные требования (обязательные)

1. Все изменения — **аддитивные**: существующие YARP-маршруты, HMAC-токены, встроенные MCP-серверы, вкладки и роутинг работают без изменений поведения.
2. Прогон существующих тестов ядра (`ClaudeHomeServer.Tests`) — зелёный.
3. Ручной smoke: OnlyOffice/drawio/forgejo-прокси, вход пользователя, одна сессия Claude со встроенными MCP.

## Вне объёма этого ТЗ

- Любая доменная логика бюджета (парсеры, категоризация, отчёты) — сторона заказчика.
- Hot-plug модулей без рестарта, WebSocket через gateway, маркетплейс модулей — вне scope v1.
- Понятие «семья»/членство — ядро отдаёт только identity (`sub`, `name`); доменная авторизация — в модуле.
