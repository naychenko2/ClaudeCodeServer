# Дизайн-аудит: Каркас рабочей области, навигация, список сессий, статусы, адаптив

Область: WorkspacePage (layout, breakpoints, сплиттеры, dock чата, мобильная навигация),
SessionList, StatusBadge, ConnectionStatus, EmptyState, NewSessionDialog, EditSessionDialog.

## Эталон (дизайн-система)

Источник: `frontend/src/lib/design.ts` (токены `C`, `BTN`) + секция «Дизайн-система» в `CLAUDE.md`.

Цвета (`C`):
- Фоны: `bgMain #F4F0E8`, `bgPanel #EDE7DC`, `bgCard #FBF8F2`, `bgWhite #FFFFFF`, `bgSelected #E8E1D4`
- Текст: `textHeading #2A251F`, `textPrimary #39332B`, `textSecondary #756B5E`, `textMuted #9A8F7E`
- Акцент: `accent #D97757`, `accentLight #F4ECE1`, `accentMuted #EAD3C5`
- Границы: `border #E0D7C8`, `borderLight #E8E1D4`
- Статусы: `success #5E8B4E`/`successBg #E9F1E8`/`successText #3F7A4F`, `warning #C9923E`/`warningBg #FBEFE0`/`warningText #8A6A28`, `danger #B4452F`/`dangerBg #FBF1EC`/`dangerText #B4452F`, `info #3E7CA6`/`infoBg #E7EFF5`

Кнопки (`BTN`): primary = `accent` фон + текст `#FBF8F2`, radius 11; secondary = `bgPanel`; ghost = border `C.border`, radius 9; danger = `C.danger` (#B4452F).

Правила: только inline-стили (без Tailwind/CSS-modules). Шрифты: PT Serif (заголовки), Hanken Grotesk (UI), JetBrains Mono (код/пути). Акцент — `#D97757`.

**Главное наблюдение аудита:** почти весь каркас написан хардкод-хексами в обход токенов `C`. Многие значения близки к токенам, но систематически расходятся (см. W1) — это создаёт несколько слегка разных «оттенков одного и того же» по приложению и ломает единый источник правды. Это сквозная проблема, ниже — конкретика.

### [W1] Тотальный обход токенов `C`: дубли и расхождения цветов панелей/границ
- **Severity:** P2
- **Режим:** все
- **Где:** WorkspacePage.tsx (почти все стили), SessionList.tsx, ConnectionStatus.tsx
- **Проблема:** Каркас не импортирует `C` из `design.ts` вообще — все цвета забиты строками. При этом значения не совпадают с токенами:
  - Граница сайдбара и хедеров — `#DDD4C4` (WorkspacePage:207, 232, 247, 288 и др.). В системе границы это `border #E0D7C8`. Получается третий «почти такой же» серо-бежевый.
  - Фон таб-свитчера — `#E1D9CA` (WorkspacePage:187, 260). Нет такого токена; ближайший `bgSelected #E8E1D4`.
  - Фон футера проекта — `#E7E0D2` (WorkspacePage:232, 288). Нет токена.
  - Сплиттеры — `#D4CFC4` / hover `#BDB7AC` / ручка `#A8A09A` (WorkspacePage:329-333 и др.). Нет токенов.
  - Десктоп-фон обёртки — `#F4F0E8` инлайном (WorkspacePage:323) дублирует `bgMain`; мобильная обёртка — `#EDE7DC` (244) дублирует `bgPanel`.
  - Текст-вторичный встречается тремя значениями: `#9A8F7E` (= `textMuted`), `#8A8070` (не токен, WorkspacePage:253,300,317), `#756B5E` (= `textSecondary`). Используются вперемешку.
- **Предложение:** Свести каркас на токены `C`. Привести `#DDD4C4`→`C.border`, обёртки фонов→`C.bgMain`/`C.bgPanel`, вторичный текст→`C.textMuted`/`C.textSecondary`. Для не покрытых ролей (фон таб-свитчера, фон футера, цвета сплиттера) — либо добавить именованные токены в `design.ts`, либо переиспользовать существующие. Сейчас невозможно поменять «границу» в одном месте.

### [W2] Сплиттеры: слишком тёмные и контрастные для спокойной палитры
- **Severity:** P2
- **Режим:** desktop
- **Где:** WorkspacePage.tsx:329, 356, 379 (полоса `#D4CFC4`), ручки `#A8A09A` (333, 360, 386); скрин da-ws-split-desktop.png
- **Проблема:** Полоса сплиттера 5px залита сплошным `#D4CFC4` — это заметно темнее границ панелей (`#E0D7C8`/`#DDD4C4`) и читается как «толстая серая колонна» между панелями, особенно в split-режиме (видно на da-ws-split-desktop.png — вертикальная полоса контрастнее всех остальных линий интерфейса). Ручка `#A8A09A` ещё темнее. В макете разделители — тонкие и тихие.
- **Предложение:** Сделать фон сплиттера в покое равным фону/границе панели (`C.border` или прозрачный), а явный «грип» (ручку) показывать только на hover. Сейчас грип виден всегда и привлекает внимание сильнее контента. Полосу 5px оставить как hit-area, но визуально утончить до 1px линии в покое.

### [W3] Hit-area сплиттеров 5px — на грани комфортного захвата
- **Severity:** P3
- **Режим:** desktop
- **Где:** WorkspacePage.tsx:329 (`flex: '0 0 5px'`), 356, 379 (`height: 5`)
- **Проблема:** 5px шириной — узкая зона для попадания мышью, особенно для вертикального ресайза дока. Курсор `col-resize`/`row-resize` корректно стоит, но цель маленькая.
- **Предложение:** Расширить интерактивную зону до ~8-10px (можно за счёт прозрачного padding/псевдо-зоны), сохранив визуальную тонкость линии. Это стандарт для перетаскиваемых разделителей.

### [W4] Мобильный чат: две шапки подряд — дублирование и потеря вертикали

- **Режим:** mobile
- **Где:** WorkspacePage.tsx:246-276 (внешняя шапка с back+табами) + ChatPanel ChatHeaderBar (имя чата + хлебные крошки); скрин da-ws-chat-mobile.png

### [W6] Авто-создание «Чат #1» при входе в проект — слот занимается без действия
- **Severity:** P2
- **Режим:** все
- **Где:** SessionList.tsx:47-52 (если список пуст и онлайн — `create` автоматически)

Авто-старт процесса откладывать до первого сообщения пользователя. Если продукт-требование «всегда есть активный чат» — хотя бы не стартовать процесс до ввода.
