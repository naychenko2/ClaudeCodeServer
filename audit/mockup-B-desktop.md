Сверка задумки дизайнера (`Claude Code Desktop.dc.html`, окно 1440×884) с реализацией
десктопной раскладки ClaudeCodeServer (`frontend/src/`). Фокус — **задумка**, а не токены.

Контекст-скидки (учтены, не считаются багами): тайтлбар ОС с traffic-light неприменим к
браузерному вебу; дерево файлов забито PNG — артефакт теста; `da-projects-empty` снят некорректно.

### [MB1] Свёрнутый док чата (focus-режим) показывает превью Claude, а не самый свежий ответ
- **Severity:** P3
- **Раскладка/экран:** Фокус-режим (файл во весь экран + свёрнутый док-полоса)
- **Задумано (макет):** Свёрнутая полоса дока (58px) — аватар + текст последнего **ответа Claude**
  («Готово — добавил декоратор и обработку 429. Запустить тесты?») + pill-поле «Ответить…» (ширина
  300px, круглая акцентная кнопка отправки) + кнопка «развернуть» (▲) в оранжевом квадрате с
  рамкой `#E6C9B8`.
- **Реализовано:** `ChatPanel` `dockMode==='collapsed'` (полоса 56px): аватар + превью (последний
  `text` ИЛИ `user_message` — берётся любой, в т.ч. собственное сообщение пользователя) + узкий
  input 180px без pill-обводки + круглая кнопка + кнопка-шеврон «развернуть» **без** оранжевого
  фона/рамки (просто иконка `#756B5E`).
- **Расхождение:** превью может показать реплику пользователя вместо ответа Claude; поле ввода уже
  и проще задуманного; кнопка «развернуть» потеряла акцентное оформление-«призыв».
- **Рекомендация:** в превью брать только последний `kind==='text'` (ответ Claude); расширить input
  до ~300px и стилизовать как pill; вернуть акцентный квадрат на кнопку развернуть.

### [MB2] Нумерация строк при просмотре файла отсутствует
- **Severity:** P2
- **Раскладка/экран:** Split (правая колонка 600px) и Фокус-режим, сегмент «Файл»
- **Задумано (макет):** Просмотр файла — две колонки: слева gutter с номерами строк
  (`{{ lineNumbers }}`, цвет `#C4BBA9`, ширина ~16–18px паддинга), справа код с подсветкой.
- **Реализовано:** `FileViewer` рендерит код через `react-syntax-highlighter` **без**
  `showLineNumbers` — номеров строк нет вообще. Markdown рендерится отдельным `MarkdownViewer`
  (это улучшение, не баг).
- **Расхождение:** ключевой элемент «читалки кода» из макета (нумерация строк) не реализован.
- **Рекомендация:** включить `showLineNumbers` у `SyntaxHighlighter` с gutter-стилем под токены.

### [MB3] Diff упрощён — нет номеров строк и hunk-заголовков `@@`
- **Severity:** P2
- **Раскладка/экран:** Split и Фокус-режим, сегмент «Diff»
- **Задумано (макет):** Diff — построчная сетка: gutter с номером строки / `+` / `−` (цветной фон
  `#EAF3E7` / `#F8E7E1`), затем строка; сверху hunk-заголовок `@@ -1,4 +1,5 @@` на голубом фоне
  `#EEF2F6`. То есть «настоящий» дифф-вьюер с разметкой.
- **Реализовано:** `FileViewer` (tab==='diff') рендерит сырой текст `git diff` в `<pre>`: каждая
  строка подкрашивается по первому символу (`+`/`-`), но **нет** колонки номеров строк, нет
  выделенного hunk-заголовка (`@@` идёт как обычная серая строка), цвета зелёный/красный иные
  (`#E8F5E9`/`#FFEBEE` вместо макетных).
- **Расхождение:** дифф выглядит как «покрашенный textatext», а не структурированный side-gutter
  diff из макета.
- **Рекомендация:** распарсить hunks, вынести номера старой/новой строки в gutter, оформить `@@`
  как отдельный заголовок-плашку.

### [MB4] Левый рельс: лишний блок «логотип + название», иные подписи вкладок
- **Severity:** P3
- **Раскладка/экран:** Все рабочие раскладки (рельс слева)
- **Задумано (макет):** Рельс начинается **сразу** с сегмент-переключателя «Сессии / Файлы»
  (padding `18px 16px 14px`), без логотипа продукта внутри панели. Низ рельса — пикер проекта
  (см. MB6).
- **Реализовано:** `WorkspacePage` добавляет над переключателем строку «лого 28px + Claude Code
  Server» (PT Serif 18px). Вкладки названы **«Чаты / Файлы»**, а в макете — **«Сессии / Файлы»**.
- **Расхождение:** появился незадуманный заголовок-лого в рельсе; терминология «Сессии» заменена
  на «Чаты» (расходится с CLAUDE.md и протоколом, где это сессии).
- **Рекомендация:** решить осознанно — либо убрать лого из рельса (как в макете), либо принять как
  ребрендинг; согласовать «Чаты» vs «Сессии» по всему продукту.

### [MB5] Picker проектов реализован как отдельная страница, а не оверлей поверх рабочего окна
- **Severity:** P2
- **Раскладка/экран:** Экран «Проекты»
- **Задумано (макет):** «Проекты» — **оверлей** `position:absolute; inset:0; z-index:25` поверх того
  же окна 1440×884 (рабочая раскладка остаётся «под» ним). Открывается кликом по пикеру в низу
  рельса; есть состояние «Новый проект» внутри того же оверлея и empty-state «Пока нет проектов».
- **Реализовано:** отдельная страница `ProjectListPage` (`onBack`), монтируется вместо
  `WorkspacePage`. Внутри есть поиск проектов (его нет в макете), счётчика сессий справа
  (`p.sessionsLabel` «3 сессии») — **нет**.
- **Расхождение:** иная навигационная модель (страница vs оверлей-пикер из рельса) и отсутствует
  счётчик сессий у карточки проекта; добавлен незадуманный поиск.
- **Рекомендация:** допустимо как веб-адаптация, но стоит вернуть счётчик сессий на карточку; при
  желании приблизить — открывать список как оверлей из футера рельса.

### [MB6] Пикер проекта в футере рельса не открывает выбор проекта
- **Severity:** P2
- **Раскладка/экран:** Футер левого рельса (рабочая раскладка)
- **Задумано (макет):** Нижний блок рельса (`openPicker`) — кнопка-пикер: иконка-точка + имя
  проекта + путь + шеврон «вверх/вниз» (↕), по клику открывает **список проектов** (оверлей MB5),
  т.е. быстрая смена проекта не выходя из рабочего окна.
- **Реализовано:** футер рельса (`ConnectionStatus` + шеврон ↕) по клику вызывает `onBack` —
  просто возвращает к списку проектов (размонтирует workspace). Это совпадает по результату, но
  семантика «пикер на месте» утеряна; плюс блок выполняет двойную роль индикатора соединения.
- **Расхождение:** «переключатель проекта» выродился в кнопку «назад».
- **Рекомендация:** ок для текущей навигации; если хотим UX макета — открывать пикер-оверлей вместо
  полного возврата.

### [MB7] Шапка чата: задуманы «статус-пилюля активна» + подпись «обновлено только что», в реализации — иначе
- **Severity:** P3
- **Раскладка/экран:** Раскладка «только чат» / split — шапка чата
- **Задумано (макет):** Заголовок сессии (17px) + строка `{{ projectName }} · обновлено только что`;
  справа — статус-пилюля «● активна» (зелёная, `#E9F1E8`).
- **Реализовано:** `ChatHeaderBar` — заголовок + строка `project · mode · model` (моноширинный
  тех-формат); статус-пилюли «активна» в шапке **нет** (есть кнопка настроек-шестерёнка и кнопка
  «Остановить» при ожидании). Статус показывается только в карточке списка сессий.
- **Расхождение:** в шапке нет визуального индикатора состояния сессии; подзаголовок несёт
  тех-данные вместо «обновлено …».
- **Рекомендация:** добавить статус-пилюлю в правую часть шапки (как в макете и в развёрнутом доке —
  там она, к слову, тоже задумана, но в реализации дока отсутствует).

### [MB8] Сегмент Файл/Diff в split-режиме: набор кнопок шапки файла расходится с макетом
- **Severity:** P3
- **Раскладка/экран:** Split (правая файловая колонка)
- **Задумано (макет):** В split-шапке файла кнопок действий три: «фокус» (развернуть), «удалить»,
  «закрыть». Кнопки правки/сохранения в split **нет** (правка — только в фокус-режиме).
- **Реализовано:** `FileViewer` в любом режиме показывает богатый набор: «Откатить» (если diff),
  «Править/карандаш», иконка облака (синхронизация офлайн), «Скачать» (для бинарных), expand/split,
  «Удалить», «Закрыть». Это функционально шире макета.
- **Расхождение:** не упрощение, а наоборот — расширение; визуально шапка плотнее задуманной.
  Облако-синхронизация и скачивание — фичи, которых в макете нет вовсе (осознанное расширение).
- **Рекомендация:** принять как осознанный апгрейд; при желании скрывать кнопку «Править» в
  split-колонке, оставив её только в фокусе (как задумано).

### [MB9] Empty-state чата: заголовок «Чем помочь?» вместо задуманного «живого» первого экрана
- **Severity:** P3
- **Раскладка/экран:** Раскладка «только чат», пустая сессия
- **Задумано (макет):** Прототип не содержит отдельного empty-state «Чем помочь?» — стартовый чат
  сразу показывает поток (session_started, сообщения, план, diff, permission). Подсказок-чипов в
  макете нет.
- **Реализовано:** `ChatPanel` рисует центрированный empty-state: лого + «Чем помочь?» +
  «Опишите задачу или начните с подсказки» + два чипа («Объясни структуру проекта», «Найди и почини
  падающие тесты»).
- **Расхождение:** добавлен незадуманный экран-онбординг. Это разумное UX-дополнение, но это
  отступление от макета.
- **Рекомендация:** оставить — улучшает первый запуск; зафиксировать как намеренное расширение.

### [MB10] Композер: «режим» вынесен в выпадающее меню вместо встроенной пилюли с подписью
- **Severity:** P3
- **Раскладка/экран:** Композер (раскладка «только чат»)
- **Задумано (макет):** В строке композера — пилюля режима с иконкой-молнией и подписью
  (`{{ modeLabel }}`), по клику открывается меню «Режим Claude» с тремя пунктами (Авто / План /
  Спросить) и описаниями. Плюс иконка вложения, микрофон, круглая кнопка отправки.
- **Реализовано:** `Composer` (по скриншотам) — иконка «+», поле «Спросите Claude…», пилюля
  «⚡ Авто», микрофон, кнопка отправки. В целом совпадает. (Меню режима/вложений детально не
  отсняты — соответствие вероятно близкое.)
- **Расхождение:** минимальное; пункт оставлен для полноты — подтвердить, что меню режима содержит
  описания пунктов, как в макете.
- **Рекомендация:** сверить тексты описаний режимов с макетом («Принимать правки автоматически» /
  «Сначала показать план» / «Подтверждать каждое действие»).

### [MB11] Login: подзаголовок-футер «Соединение шифруется end-to-end» не из макета; кнопка по умолчанию выглядит disabled
- **Severity:** P3
- **Раскладка/экран:** Логин (во весь лист)
- **Задумано (макет):** Логин: лого 54px, заголовок, подзаголовок «Укажите адрес сервера и
  API-ключ…», два поля, тоггл «Сохранить ключ в связке устройства», кнопка «Подключиться →».
  Подписи-футера про шифрование **нет**.
- **Реализовано:** добавлена строка «Соединение шифруется end-to-end» под кнопкой; кнопка
  «Подключиться» в пустом состоянии отрисована приглушённо-розовой (выглядит как disabled, хотя в
  макете она всегда насыщенно-акцентная `#D97757`). Поля и тоггл соответствуют.
- **Расхождение:** незадуманная строка про шифрование; иной вид кнопки в пустом состоянии.
- **Рекомендация:** убрать/проверить уместность строки про end-to-end (может вводить в
  заблуждение); кнопку оставлять акцентной либо явно показывать disabled-состояние.
