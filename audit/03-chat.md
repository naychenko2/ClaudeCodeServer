# Дизайн-аудит: Чат (ChatPanel, Composer, MarkdownViewer)

## Эталон (дизайн-система)

Источник: `frontend/src/lib/design.ts` (токены `C`, `BTN`) и секция «Дизайн-система» в `CLAUDE.md`.

**Акцент:** `accent #D97757` (основной цвет кнопок и активных состояний), `accentLight #F4ECE1`, `accentMuted #EAD3C5`.

**Фоны:** `bgMain #F4F0E8` (страницы), `bgPanel #EDE7DC` (боковые панели/заголовки), `bgCard #FBF8F2` (карточки), `bgWhite #FFFFFF`, `bgSelected #E8E1D4`.

**Текст:** `textHeading #2A251F`, `textPrimary #39332B`, `textSecondary #756B5E`, `textMuted #9A8F7E`.

**Границы:** `border #E0D7C8`, `borderLight #E8E1D4`.

**Статусы (есть Bg/Text-варианты):**
- success `#5E8B4E` / successBg `#E9F1E8` / successText `#3F7A4F`
- warning `#C9923E` / warningBg `#FBEFE0` / warningText `#8A6A28`
- danger `#B4452F` / dangerBg `#FBF1EC` / dangerText `#B4452F`
- info `#3E7CA6` / infoBg `#E7EFF5`

**Diff:** diffAddBg `#E8F5E9` / diffAddText `#1B5E20`, diffRemBg `#FFEBEE` / diffRemText `#B71C1C`.

**Шрифты:** PT Serif (заголовки), Hanken Grotesk (UI), JetBrains Mono (код/пути/diff).

**Кнопки (BTN):** primary `bg accent, color #FBF8F2, radius 11, padding 11×20`; secondary `bg bgPanel, color textSecondary`; ghost `border 1px border, radius 9`; danger `bg danger, color #FBF8F2, radius 9`.

**Правило:** только inline-стили; цвета — только из палитры.

**Важная оговорка по аудиту:** дизайн-система проекта (`design.ts` + `CLAUDE.md`) отличается от обобщённой палитры в системном промпте (там `--success #27AE60`, `--danger #C0392B`, `--badge-modified #E67E22` и т. п.). За эталон беру **`design.ts`** — это реальный источник истины кодовой базы. Все расхождения ниже меряю по токенам `C`.

## Находки

### [C1] Хардкод-цвета вместо токенов `design.ts` по всему чату
- **Severity:** P2
- **Режим:** все
- **Где:** `ChatPanel.tsx` целиком (нет `import { C, BTN }`), `Composer.tsx` целиком, `MarkdownViewer.tsx` целиком
- **Проблема:** Ни один из трёх компонентов чата не импортирует `C`/`BTN` — все цвета прописаны строковыми литералами (`'#D97757'`, `'#2A251F'`, `'#9A8F7E'`, `'#E7E0D2'`, `'#EDE7DA'` и т. д.). Это нарушает правило «цвета только из палитры через токены» и приводит к расхождениям (см. C2, C3, C4): один и тот же по смыслу цвет записан разными значениями в разных местах. Любая правка палитры в `design.ts` не дойдёт до чата.
- **Предложение:** Завести `import { C, BTN } from '../lib/design'` и заменить литералы на токены. Это устранит и точечные расхождения C2–C4 заодно.

### [C2] Статусные цвета file_changed и ошибок не совпадают с токенами
- **Severity:** P2
- **Режим:** все (видно на `da-ws-chat-typing-desktop.png`, `da-ws-chat-answer-desktop.png` — карточки sessions.json / history.json)
- **Где:** `ChatPanel.tsx:843-848` (счётчики +N/−N), `ChatPanel.tsx:757` (статус инструмента), `ChatPanel.tsx:866` (кнопка «Откатить»), `ChatPanel.tsx:887-907` (блок error)
- **Проблема:** Цвета добавления/удаления и ошибок взяты «с улицы», а не из токенов:
  - `+{item.added}` → `#27AE60` (хардкод), а в `design.ts` зелёный — `success #5E8B4E` / `successText #3F7A4F`. `#27AE60` не из палитры.
  - `-{item.removed}` → `#C0392B`, кнопка «Откатить» → `#C0392B`, ошибка инструмента → `#C0392B`, блок error фон/текст/border → `#FDECEA`/`#C0392B`/`#F5C6CB`. В палитре красный — `danger #B4452F`, `dangerBg #FBF1EC`. `#C0392B`/`#FDECEA`/`#F5C6CB` не из палитры.
  - Замечание по смыслу: для счётчиков diff в карточке логичнее брать diff-токены (`diffAddText #1B5E20` / `diffRemText #B71C1C`) — они и предназначены для +/− в диффах. Сейчас же используется третий набор зелёного/красного.
- **Предложение:** `+N` → `C.diffAddText` (или `C.successText`), `−N` → `C.diffRemText`; «Откатить»/error → `C.danger`, фон error → `C.dangerBg`, бордер error → производный от `C.danger`. Привести к единому набору красного/зелёного во всём чате.

### [C3] Инлайн-код в ответе: фон и цвет текста не из палитры
- **Severity:** P2
- **Режим:** все (хорошо видно на `da-ws-chat-answer-desktop.png` — `backend/`, `data/projects.json`, `#D97757` и пр.)
- **Где:** `ChatPanel.tsx:178` (инлайн-код в `MarkdownContent`), дублируется в `MarkdownViewer.tsx:41`
- **Проблема:** Инлайн-код стилизован как `background: '#EDE7DA'`, `color: '#5A3322'` (в MarkdownViewer — `#5C3D2E`). Ни `#EDE7DA`, ни `#5A3322`/`#5C3D2E` нет в палитре. По смыслу фон близок к `bgPanel #EDE7DC`/`borderLight #E8E1D4`, а цвет текста — кастомный коричневый. Плюс два разных значения цвета текста кода в двух компонентах (`#5A3322` vs `#5C3D2E`) — несогласованность одного и того же элемента.
- **Предложение:** Единый стиль инлайн-кода: фон `C.bgPanel` (или `C.borderLight`), текст — один токен (например `C.textPrimary` или зафиксированный «код-коричневый», добавленный в `design.ts`). Синхронизировать `MarkdownContent` и `MarkdownViewer` — это один и тот же визуальный элемент.

### [C4] Граница-разделитель `#E7E0D2` вместо токена border
- **Severity:** P3
- **Режим:** все
- **Где:** `ChatPanel.tsx:65` (низ хедера), `ChatPanel.tsx:446,571` (верх композера), `ChatPanel.tsx:736` (рамки tool_use)
- **Проблема:** Разделители используют `#E7E0D2`, которого нет в палитре. Рядом эталонные `border #E0D7C8` и `borderLight #E8E1D4`. Третий близкий оттенок границы плодит визуальный шум и расходится с остальным UI (хедер/панели проекта используют `#E7E0D2` тоже, но это всё равно не токен).
- **Предложение:** Заменить на `C.border` или `C.borderLight`.

### [C5] Длинный ответ растягивается на всю ширину в dock-режиме
- **Severity:** P2
- **Режим:** desktop (dock expanded)
- **Где:** `ChatPanel.tsx:418-443` (dock expanded) — нет ограничителя `maxWidth: 760`, который есть в полноэкранном режиме (`ChatPanel.tsx:493`)
- **Проблема:** В полноэкранном чате контент обёрнут в `maxWidth: 760` по центру — длина строки читаема (видно на `da-ws-chat-answer-desktop.png`: текст не доходит до правого края). В развёрнутом доке такой обёртки нет: список сообщений идёт на всю ширину `padding: '12px 16px'`. При широком доке строки ответа будут чрезмерно длинными (>90 симв.), что бьёт по читаемости. Композер в доке тоже без ограничителя ширины.
- **Предложение:** В dock-expanded обернуть и список сообщений, и композер в тот же `maxWidth: 760; margin: 0 auto`, как в полноэкранном режиме. Либо вынести обёртку в общий компонент, чтобы не дублировать.

### [C6] result-строка «✓ N шагов · Xс»: грамматика и слабая иерархия
- **Severity:** P3
- **Режим:** все (видно `da-ws-chat-answer-desktop.png` — «✓ 1 шагов · 12.7с»)
- **Где:** `ChatPanel.tsx:877-885`
- **Проблема:** Два момента:
  1. «1 шагов» — нет согласования числительного («1 шаг», «2 шага», «5 шагов»). Мелкая, но заметная небрежность прямо в кадре.
  2. Плашка `background: '#E8E2D6'` — `#E8E2D6` нет в палитре (близко к `bgSelected #E8E1D4`); галочка `✓` и крест `✗` рисуются текстом и не несут статусного цвета — успех/ошибка визуально неразличимы (оба серые `#8A8070`). Для `subtype !== 'success'` логично подсветить крест в `C.danger`.
- **Предложение:** Добавить плюрализацию шагов; фон плашки → `C.bgSelected`; для ошибки красить маркер в `C.dangerText`, для успеха — оставить нейтральным либо `C.successText`.

### [C7] Индикаторы загрузки оформлены тремя разными способами
- **Severity:** P2
- **Режим:** все
- **Где:** `ChatPanel.tsx:435-440,561-566` («ожидаю ответа …»), `Composer.tsx:203-209` («Claude печатает…»), `ChatPanel.tsx:739` (`ToolSpinner` у инструмента)
- **Проблема:** Три разных индикатора «идёт работа» в одном экране:
  - «ожидаю ответа …» — серый текст 12px с CSS-анимацией `.dots` + отдельный «…».
  - «Claude печатает…» — внутри композера, italic 14px `#9A8F7E` + `ThreeDots` (класс `.composer-dot`).
  - спиннер инструмента — крутилка `.tool-spinner`.
  Тексты дублируют смысл («ожидаю ответа» в ленте и «Claude печатает» в композере показываются одновременно, см. `da-ws-chat-typing-desktop.png`), но выглядят по-разному (разные размеры, цвета, анимации). Нет единого визуального языка ожидания.
- **Предложение:** Унифицировать: оставить один индикатор-источник (логичнее «Claude печатает…» в композере, раз он уже там), а дублирующий «ожидаю ответа» в ленте либо убрать, либо привести к тому же стилю (размер/цвет/анимация). Цвет точек/текста — `C.textMuted`.

### [C8] Кнопки file_changed «Открыть»/«Откатить» — не из BTN, мелкие тач-таргеты
- **Severity:** P3
- **Режим:** все (особенно mobile)
- **Где:** `ChatPanel.tsx:850-872`
- **Проблема:** Кнопки заданы вручную (`padding: '4px 10px'`, `fontSize: 12`) вместо `BTN.ghost`/`BTN.secondary`. Высота кнопки ~25px — на мобильном это меньше рекомендованных 44px тач-таргета (карточка file_changed показывается и на mobile — `da-ws-chat-mobile.png`). Цвет «Откатить» — `#C0392B` (см. C2). Бордеры разные: «Открыть» — `#E8E1D4`, «Откатить» — `#E0D8CC` (последнего нет в палитре).
- **Предложение:** Использовать `BTN.ghost` для «Открыть» и danger-вариант для «Откатить» (текст `C.danger`); на мобильном увеличить вертикальный padding до тач-таргета ≥40px. Унифицировать цвет бордера (`C.borderLight`).

### [C9] Кнопки-чипы подсказок в пустом чате — не из дизайн-системы
- **Severity:** P3

### [C10] Аватар «CLAUDE» и логотипы заданы магическими размерами без токенов
