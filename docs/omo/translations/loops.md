# Шаблоны циклов «до готово»

Протокольные константы (из `constants.ts`, справочно): паттерн завершения `<promise>(.*?)</promise>`, лимит итераций по умолчанию — 100, в режиме ultrawork — 500, промис завершения по умолчанию — `DONE`, промис верификации Оракула — `VERIFIED`.

---

## Шаблон Ralph-цикла (RALPH_LOOP_TEMPLATE)

Ты запускаешь Ralph-цикл — самоссылающийся цикл разработки, который идёт до завершения задачи.

### Как работает Ralph-цикл

1. Ты работаешь над задачей непрерывно
2. Когда считаешь, что задача ПОЛНОСТЬЮ завершена, выведи: `<promise>{{COMPLETION_PROMISE}}</promise>`
3. Если ты не вывел промис, цикл автоматически подкинет следующий промпт для продолжения
4. Максимум итераций: настраивается (по умолчанию 100)

### Правила

- Сосредоточься на полном, а не частичном завершении задачи
- Не выводи промис завершения, пока задача не сделана по-настоящему
- Каждая итерация должна давать осмысленный прогресс к цели
- Застрял — пробуй другие подходы
- Отслеживай прогресс через todo-инструменты

### Условия выхода

1. **Завершение**: выведи свой тег-промис завершения, когда всё полностью готово
2. **Лимит итераций**: цикл автоматически останавливается на пределе
3. **Отмена**: пользователь выполняет команду `/cancel-ralph`

### Твоя задача

Разбери аргументы ниже и начинай работать над задачей. Формат:
`"описание задачи" [--completion-promise=TEXT] [--max-iterations=N] [--strategy=reset|continue]`

Промис завершения по умолчанию — «DONE», максимум итераций по умолчанию — 100.

## Шаблон ULTRAWORK-цикла (ULW_LOOP_TEMPLATE)

Ты запускаешь ULTRAWORK-цикл — самоссылающийся цикл разработки, который идёт до верифицированного завершения.

### Как работает ULTRAWORK-цикл

1. Ты работаешь над задачей непрерывно
2. Когда считаешь работу завершённой, выведи: `<promise>{{COMPLETION_PROMISE}}</promise>`
3. Это ещё НЕ завершает цикл. Система потребует верификацию Оракулом
4. Цикл заканчивается только после того, как система подтвердит, что Оракул верифицировал результат
5. Лимит итераций — 500 в режиме ultrawork, 100 в обычном режиме

### Правила

- Сосредоточься на полном завершении задачи
- После вывода промиса завершения выполни верификацию Оракулом, когда система это потребует
- Не считай DONE финальным завершением, пока Оракул его не верифицировал

### Условия выхода

1. **Верифицированное завершение**: Оракул верифицирует результат, и система это подтверждает
2. **Отмена**: пользователь выполняет `/cancel-ralph`

### Твоя задача

Разбери аргументы ниже и начинай работать над задачей. Формат:
`"описание задачи" [--completion-promise=TEXT] [--strategy=reset|continue]`

Промис завершения по умолчанию — «DONE».

## Continuation-сообщения цикла

Эти сообщения система подкидывает в чат между итерациями (в оригинале каждое начинается с системного префикса `[SYSTEM DIRECTIVE: OH-MY-OPENCODE - ...]`; в режиме ultrawork всё сообщение дополнительно предваряется словом `ultrawork`). Плейсхолдеры подставляются из состояния цикла: `{{ITERATION}}` — номер итерации, `{{MAX}}` — лимит (или «unbounded»), `{{PROMISE}}` — текущий промис завершения, `{{INITIAL_PROMISE}}` — исходный промис, `{{PROMPT}}` — исходная задача.

### Обычное продолжение (CONTINUATION_PROMPT)

```
[СИСТЕМНАЯ ДИРЕКТИВА - RALPH-ЦИКЛ {{ITERATION}}/{{MAX}}]
Продолжай. Когда закончишь, выведи <promise>{{PROMISE}}</promise>.
{{PROMPT}}
```

### Требование верификации ultrawork (ULTRAWORK_VERIFICATION_PROMPT)

```
[СИСТЕМНАЯ ДИРЕКТИВА - ВЕРИФИКАЦИЯ ULTRAWORK-ЦИКЛА {{ITERATION}}/{{MAX}}]

Ты уже вывел <promise>{{INITIAL_PROMISE}}</promise>. Это ещё НЕ завершает цикл.

ТРЕБУЕТСЯ СЕЙЧАС:
- Вызови Оракула — делегируй субагенту-Оракулу синхронно (не в фоне)
- Попроси Оракула проверить, действительно ли исходная задача завершена
- Включи исходную задачу в запрос Оракулу
- Явно скажи Оракулу проверять скептически и критически и искать причины, по которым задача может быть всё ещё не завершена или сделана неверно
- Система сама проинспектирует сессию Оракула на предмет результата верификации
- Если Оракул не верифицировал — продолжай чинить задачу и не считай её завершённой

Исходная задача:
{{PROMPT}}
```

### Провал верификации (ULTRAWORK_VERIFICATION_FAILED_PROMPT)

```
[СИСТЕМНАЯ ДИРЕКТИВА - ВЕРИФИКАЦИЯ ULTRAWORK-ЦИКЛА ПРОВАЛЕНА {{ITERATION}}/{{MAX}}]

Оракул не вывел <promise>VERIFIED</promise>. Верификация провалена.

ТРЕБУЕТСЯ СЕЙЧАС:
- Верификация провалена. Чини задачу, пока ревью Оракула не будет удовлетворено
- Оракул не лжёт. Считай результат верификации истиной в последней инстанции
- Не заявляй завершение раньше времени и не спорь с проваленной верификацией
- Исправив оставшиеся проблемы, снова запроси ревью Оракула — делегируй субагенту-Оракулу синхронно
- Включи исходную задачу в запрос Оракулу и скажи ему проверять скептически и критически
- Только когда работа снова готова к ревью, выведи: <promise>{{PROMISE}}</promise>

Исходная задача:
{{PROMPT}}
```

## Шаблон отмены (CANCEL_RALPH_TEMPLATE)

Отмени активный сейчас Ralph-цикл.

Это:
1. Остановит продолжение цикла
2. Очистит файл состояния цикла
3. Позволит сессии завершиться нормально

Проверь, активен ли цикл, и отмени его. Сообщи пользователю результат.

## Адаптации

- Рабочий тег `<promise>...</promise>` и все плейсхолдеры (`{{COMPLETION_PROMISE}}`, `{{ITERATION}}`, `{{MAX}}`, `{{PROMISE}}`, `{{INITIAL_PROMISE}}`, `{{PROMPT}}`) сохранены дословно — это протокол детектора завершения (regex `<promise>(.*?)</promise>` из constants.ts). Литерал `VERIFIED` тоже не переведён — по нему система распознаёт вердикт Оракула.
- Префикс `[SYSTEM DIRECTIVE: OH-MY-OPENCODE - ...]` в continuation-сообщениях заменён на `[СИСТЕМНАЯ ДИРЕКТИВА - ...]` — в оригинале это константа-маркер их обвязки (SYSTEM_DIRECTIVE_PREFIX), у нас обвязка своя; факт отмечен в тексте раздела.
- `task(subagent_type="oracle", load_skills=[], run_in_background=false, ...)` → «делегируй субагенту-Оракулу синхронно (не в фоне)» — вызов Task/Agent в Claude Code; Oracle = Оракул (Консультант) по карте имён.
- «Use todos to track your progress» переведено как «отслеживай прогресс через todo-инструменты» (todo-протокол Claude Code).
- Имена команд (`/cancel-ralph`), флаги аргументов (`--completion-promise`, `--max-iterations`, `--strategy=reset|continue`) и дефолт «DONE» сохранены как есть. Путь файла состояния OmO (`.omo/ralph-loop.local.md`) в переводы не включён — в шаблоне отмены оставлена нейтральная формулировка «файл состояния цикла».
- Помимо двух шаблонов запуска переведены три continuation-промпта из `continuation-prompt-builder.ts` и шаблон отмены CANCEL_RALPH_TEMPLATE (лежит в том же файле-источнике).
