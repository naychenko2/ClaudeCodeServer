# Паритет фич с Claude Code

Сравнение возможностей официального Claude Code CLI и их реализации в ClaudeCodeServer
(веб-обёртка вокруг `claude.exe` со stream-json через SignalR).

Дата сводки: 2026-06-29.

> **Обновление 2026-06-29:** реализованы 7 фич — reasoning effort, накопительный cost,
> рендер todo-листа (уже был), push-уведомления, persistence (уже был полным), отправка
> изображений как content-блоков, гранулярные permission-правила на уровне проекта.
> Статусы ниже обновлены.

**Легенда:** ✅ полная · 🟡 частичная · ❌ нет · ➖ неприменимо для веб-обёртки

> Состав фич CLI собран по знаниям + docs.anthropic.com; конкретные номера версий могут быть
> неточны. Привязка к нашему коду (файлы/строки) проверена по репозиторию.

## Неприменимо для веб-обёртки (➖)

IDE-плагины (VS Code/JetBrains), TUI, vim mode, mouse-click в fullscreen, keybindings,
sandboxing на уровне ОС, OpenTelemetry, cloud execution — это про нативный CLI/окружение,
не про прокси в браузер.

## Дорожная карта по топ-пробелам

Отсортировано по соотношению ценность / стоимость реализации в обёртке.

### 1. Гранулярные permission-правила (дёшево, высокая ценность)
Сейчас — только разовые решения + auto-allow на сессию. Добавить персистентные allow/deny
паттерны на уровне проекта/сессии.
- **Бэкенд:** хранить правила в `Session`/проекте, матчить в обработчике `PermissionRequestMessage`
  до отправки запроса в UI; авто-allow/deny по паттерну (`Bash(npm run *)`, path-globs).
- **Фронт:** простой редактор списка правил в настройках сессии.
- **Стоимость:** низкая — логика матчинга + CRUD, без изменения протокола.

### 2. Checkpoints / откат файлов (средне, высокая ценность)
Снимок изменённых файлов перед ходом → откат к точке. Самая заметная недостающая фича
для безопасной работы.
- **Бэкенд:** перед `RunTurnAsync` снимать снапшот затронутых файлов (или git stash/commit
  в служебную ветку, раз git CLI уже используется для diff/revert).
- **Фронт:** кнопка «откатить к ходу N» в ленте.
- **Стоимость:** средняя — нужно аккуратно определять набор файлов хода и хранить снапшоты.

### 3. UI управления MCP / skills / агентами (средне)
Бэкенд уже их читает — не хватает только интерфейса.
- MCP: редактор `mcp-config` (список серверов, env), рестарт сессии.
- Skills/agents: список + просмотр; создание/редактирование — опционально.
- **Стоимость:** средняя, преимущественно фронт.

### 4. Reasoning effort (дёшево)
Проброс уровня thinking в запуск (если CLI принимает соответствующий флаг/поле).
- **Фронт:** селектор уровня рядом с выбором модели.
- **Бэкенд:** прокинуть в аргументы/конфиг хода.
- **Стоимость:** низкая, если CLI поддерживает параметр.

### 5. Накопительный cost tracking + `/cost` (дёшево)
`total_cost_usd` уже парсится — суммировать по сессии/проекту и показывать счётчик.
- **Стоимость:** низкая.

### 6. Fork / branch сессии (средне)
Скопировать историю до точки в новую сессию (`--fork-session` или ручное копирование
`ChatHistory` + новый `--resume`-id).
- **Стоимость:** средняя — зависит от модели хранения истории.

### Низкий приоритет / отложить
Hooks, scheduled tasks/cron, plugins/marketplace, output styles, отправка изображений как
image-блоков, push-уведомления — отдельные подсистемы, ценность для текущего сценария ниже.
