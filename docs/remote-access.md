# Удалённый доступ к Claude Code Server

Сервер должен быть доступен **дома по WiFi** и **снаружи через мобильную сеть**.
Решение состоит из двух частей: транспорт (как дотянуться до сервера) и аутентификация
(как не пустить чужих).

```
   дом / WiFi                          мобильная сеть / интернет
       │                                        │
       ▼                                        ▼
  http://<IP>:5000                     Tailscale (VPN-mesh, пробивает CGNAT)
  (уже работает)                       https://<host>.<tailnet>.ts.net
       │                                        │
       └────────────────┬───────────────────────┘
                        ▼
            ASP.NET Core :5000 (0.0.0.0)
            [Authorize] на всех API + Hub
            доступ только по API-ключу
```

## 1. Локальный доступ (дома, по WiFi)

Уже работает из коробки — сервер слушает `0.0.0.0:5000`:

1. Узнай IP домашнего ПК: `ipconfig` → строка `IPv4-адрес` (например `192.168.1.50`).
2. С телефона в той же WiFi-сети открой `http://192.168.1.50:5000`.
3. Если не открывается — разреши порт в брандмауэре Windows:
   ```powershell
   New-NetFirewallRule -DisplayName "ClaudeCodeServer" -Direction Inbound `
       -Protocol TCP -LocalPort 5000 -Action Allow
   ```

> По HTTP PWA не устанавливается как приложение (нужен HTTPS либо localhost), но в
> браузере всё работает. Для установки PWA дома используй тот же `*.ts.net`-адрес из п. 2.

## 2. Внешний доступ (мобильная сеть) — Tailscale

Tailscale строит приватную mesh-сеть между твоими устройствами. Работает за CGNAT
(у мобильных операторов белого IP нет), трафик шифруется WireGuard, сервер виден
**только твоим устройствам**. Один и тот же адрес — и дома, и в дороге.

### Настройка

1. Зарегистрируйся на <https://tailscale.com> (можно через Google).
2. Установи Tailscale на **домашний ПК** (Windows) и **телефон**, войди в один аккаунт.
3. На ПК запусти сервер как обычно (`dotnet run --project ClaudeCodeServer` или собранный exe).
4. Узнай Tailscale-адрес ПК: иконка в трее → или `tailscale ip -4` (вид `100.x.x.x`),
   либо MagicDNS-имя `<host>.<tailnet>.ts.net`.
5. С телефона (по мобильной сети, Tailscale включён) открой `http://100.x.x.x:5000`.

### HTTPS для PWA (рекомендуется)

Чтобы PWA ставилась как приложение и работал Service Worker, нужен HTTPS. Tailscale
выдаёт валидный сертификат для MagicDNS-имени автоматически:

```powershell
# Включить MagicDNS и HTTPS-сертификаты в админке Tailscale (один раз):
#   admin console → DNS → Enable MagicDNS, Enable HTTPS

# Проксировать https://<host>.<tailnet>.ts.net → localhost:5000
tailscale serve --bg 5000
```

После этого заходи на `https://<host>.<tailnet>.ts.net` (без порта) — валидный TLS,
PWA устанавливается. Доступ остаётся приватным (только tailnet).

> ⚠️ Не используй `tailscale funnel` — он публикует сервер в открытый интернет.
> Нам это не нужно: сервер исполняет команды Claude и даёт доступ к файлам.

## 3. Аутентификация (обязательна)

Все API-эндпоинты и SignalR-хаб закрыты атрибутом `[Authorize]` — доступ только по
API-ключу.

### Откуда берётся ключ (по приоритету)

1. **Конфигурация** `Auth:ApiKey` — переменная окружения, appsettings или user-secrets:
   ```powershell
   $env:Auth__ApiKey = "мой-секретный-ключ"
   dotnet run --project ClaudeCodeServer
   ```
2. **Файл** `data/auth-key.txt` рядом с executable (или путь из `Auth:KeyFile`).
3. **Автогенерация** — если ключ нигде не задан, сервер генерирует случайный ключ,
   сохраняет в `data/auth-key.txt` и печатает в консоль при старте:
   ```
   API-ключ не был задан — сгенерирован новый и сохранён в .../data/auth-key.txt
   ВВЕДИТЕ ЭТОТ КЛЮЧ НА СТРАНИЦЕ ВХОДА:
   cck_XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
   ```

### Как клиент передаёт ключ

- Страница входа сохраняет ключ в `localStorage` (`cc_api_key`).
- REST-запросы уходят с заголовком `Authorization: Bearer <ключ>`.
- WebSocket (SignalR) передаёт ключ как `?access_token=<ключ>` (заголовок там задать нельзя).
- На ответ `401` фронт автоматически разлогинивает и возвращает на экран входа.

### Смена ключа

Удали `data/auth-key.txt` (сгенерируется новый) либо задай `Auth:ApiKey`. После смены
нужно заново ввести ключ на странице входа на всех устройствах.

## Итог

| Сценарий | Адрес |
|---|---|
| Дома, браузер | `http://<локальный-IP>:5000` |
| Дома/в дороге, PWA + HTTPS | `https://<host>.<tailnet>.ts.net` |
| В дороге, без HTTPS | `http://100.x.x.x:5000` (Tailscale IP) |

Во всех случаях нужен один и тот же API-ключ.
