# Хаб раздела «Персоны» — концепция (на утверждение)

Кликабельный макет: `docs/mockups/personas-hub.html`.

## Проблема

Центральная область раздела «Персоны», когда персона не выбрана, сейчас — голая
заглушка (иконка + заголовок + подсказка + кнопка «Новая персона»). Для главной
страницы целого раздела это слишком бедно. Готового аналога «командного хаба» для
копирования в коде нет — вкладка «Команда» в проектах устроена так же скудно
(`ProjectPersonaEmpty` в [ProjectPersonasPanel.tsx](../../frontend/src/features/personas/ProjectPersonasPanel.tsx)).
Композиция ниже спроектирована с нуля, в дизайн-системе продукта.

## Итерация 2 (правки пользователя)

1. Убран блок «Пантеон OmO» — не нужен на этой странице.
2. Добавлен таймлайн активности (вместо плоского списка «Недавние разговоры»).

## Итерация 3 (правки пользователя)

1. Добавлена **витрина команды** — сетка карточек всех персон (не дублирует
   компактный список в сайдбаре: крупнее, с описанием роли, hover-действием
   «Поговорить», плюс замыкающая плитка «Новая персона»).
2. **«Быстрые действия» и «Шаблоны» объединены** в одну компактную панель
   «Новая персона»: два действия сверху (создать с нуля / описать словами —
   строка ввода теперь раскрывается по клику, а не висит всегда) и заготовки
   чипами сразу под ними, без отдельного скролла к секции.
3. **Активность стала вторым вертикальным блоком на десктопе** — правая колонка
   рядом с основным контентом (`hub-grid`: `1fr` + `300px`, `position:sticky`),
   а не секция в общем вертикальном потоке. На узких экранах (≤1160px)
   схлопывается обратно в одну колонку.
4. Из hero убран ряд аватаров-превью — дублировал витрину, теперь текст hero
   компактнее сам по себе.

## Итерация 4 (правки пользователя)

1. **Блок переименован**: «Витрина команды» → «Твои помощники», и слово
   «команда»/«команды» убрано со всей страницы (заголовок hero, описание
   Сизифа, пункт возможностей «Команда») — это персоны и помощники, не
   «команда».
2. **«Твои помощники» подняты выше** «Новой персоны» в левой колонке —
   сначала показываем, кто уже есть, потом — как добавить ещё.
3. **Шапка объединена с «Что умеет персона»**: вместо отдельной секции
   внизу страницы — компактная карточка-список справа от hero-текста
   (иконка + короткий пункт, без описаний). Это заодно закрыло пустое место
   сверху справа, которое осталось после того, как в итерации 2 убрали ряд
   аватаров из hero (у `hub-hero` было `justify-content:space-between` на
   единственного ребёнка — второй колонки не было, вот и дыра).

## Итерация 5 (правки пользователя)

Добавлена возможность посмотреть активность подробнее: ссылка «Смотреть всё →»
в заголовке блока «Активность» открывает модалку с полной лентой (6 групп по
дням вместо 3, 12 событий вместо 6) и фильтром-чипами по конкретному
помощнику (переиспользует существующие карточки `.timeline-*`, просто в более
широком контейнере). Закрытие — крестик, клик по фону или Esc.

## Итерация 6 (правки пользователя)

1. **Попап убран.** Пользователю не понравилась модалка — заменил на разворот
   на месте: фильтр-чипы теперь встроены прямо в блок «Активность» (действуют
   сразу на всю ленту), а 3 дополнительные группы («Позавчера», «На прошлой
   неделе», «В этом месяце») скрыты в `.timeline-extra` (`max-height:0`) и
   разворачиваются по клику на «Показать ещё 6 событий» / «Свернуть» — без
   какого-либо оверлея поверх страницы.
2. **Местоимения — мужской род.** В hero-тексте «у неё есть роль… разговаривай
   с ней» → «у него есть роль… разговаривай с ним»; аналогично поправлен тост
   у шаблонов («открылась бы её карточка» → «открылся бы его профиль»).

## Итерация 7 (правки пользователя)

Разворот активности переделан ещё раз — вместо инлайн-раскрытия в узкой
колонке теперь кнопка-иконка (⤢) в шапке «Активности» переключает саму
контентную зону:

- **Свёрнуто** (по умолчанию): обычная сетка `1fr 300px` — «Твои помощники» +
  «Новая персона» слева, «Активность» узкой липкой колонкой справа.
- **Развёрнуто**: `hub-grid` переходит в `display:block`, `.hub-main`
  скрывается (`display:none`), «Активность» занимает всю контентную зону под
  шапкой (центрированная колонка до 760px). Фильтр-чипы работают как раньше.
- Внизу развёрнутой ленты — сентинел на `IntersectionObserver` (`root` —
  сам `.content`, а не окно, т.к. скроллится именно контентная зона): при
  приближении к низу подгружает ещё 2 события с показом спиннера ~550мс —
  условно «виртуальный скролл» (на деле — постраничная подгрузка в DOM,
  честной виртуализации узлов нет, но UX ощущается как бесконечная лента).
  Останавливается после 4 подгрузок с сообщением «Это все события за
  последние два месяца».

## Предложенный состав

**Hero** (во всю ширину) → двухколоночная сетка (**слева**: «Новая персона»
компактной панелью + «Витрина команды»; **справа**, зафиксирована при скролле:
«Активность») → **«Что умеет персона»** во всю ширину внизу.

1. **Hero-заголовок** — объясняет за 2 строки, что такое персона.
2. **Новая персона** — компактная панель: 2 действия (с нуля / по описанию,
   ИИ придумывает роль-характер-аватар) + заготовки чипами тут же, без отдельной
   секции.
3. **Витрина команды** — все персоны крупными кликабельными карточками
   (аватар, роль-имя, описание, «Поговорить» по hover) + плитка добавления.
4. **Активность** (правая колонка) — таймлайн по дням (Сегодня / Вчера / На
   этой неделе), в стиле `GroupTimeline` из «Что нового» (маркеры-кружочки
   единым accent-цветом). Смешивает типы событий персон — разговоры, выполненные
   задачи, новые факты в памяти, @упоминания, создание персоны.
5. **Что умеет персона** — обучающая полоса внизу (память / задачи /
   автоматизация / команда / @упоминания).

## Замечание (не блокирует, просто для сведения)

Технически «Пантеон OmO» подключается именно как ГЛОБАЛЬНЫЕ персоны
(`POST /api/personas/pantheon/connect` — «идемпотентно подключить команду
**глобально**», см. CLAUDE.md раздел «Персоны»). Так что по факту он более уместен
именно в этом разделе, а не в командах проектов. Но раз с этой страницы его решили
убрать — убрал; персоны Пантеона, если уже подключены, всё равно останутся видны в
сайдбаре (это отдельное, не тронутое здесь поведение `PersonaList.tsx`).
