# Mockup C — Tablet (планшетная раскладка)

Сверка: задумка макета `Claude Code Tablet.dc.html` (DesignSync, получен полностью, не truncated) vs реализация ClaudeCodeServer (адаптивный код, breakpoint `isTablet` 768–1100px в `WorkspacePage.tsx`).

Контекст: реализация — единый адаптивный код под все ширины, не отдельное планшетное приложение; данные демо. Скриншот `da-ws-file-tablet.png` отрендерился как крошечное искажённое превью (открыт `history.json` в мелком масштабе) и нечитаем — выводы по просмотру файла на планшете опираются на код `WorkspacePage.tsx` (строки 369–399) и `FileViewer.tsx`.

---

### [MC1] Просмотр файла на планшете: макет отдаёт центр ТОЛЬКО под файл, реализация добавляет док чата снизу
- **Severity:** P1
- **Экран:** Рабочее пространство, открыт файл (`da-ws-file-tablet.png`)
- **Задумано (макет):** Центр — это взаимоисключающий переключатель: `<sc-if value="{{ centerChat }}">` ИЛИ `<sc-if value="{{ centerFile }}">`. Когда открыт файл, центр целиком занят просмотром файла (шапка файла + тело), чата на этом экране нет вообще. Возврат к чату — через кнопку «‹ {sessionTitle}» в шапке файла (`backToChat`). Рельс слева остаётся.
- **Реализовано:** На планшете (`openFile && (fileFullscreen || isTablet)`, строки 369–399) центр делится по вертикали: сверху `FileViewer` (fullscreen), снизу — постоянный **док `ChatPanel`** с горизонтальным сплиттером и кнопкой свернуть/развернуть (`dockMode`, `chatHeight`, по умолчанию развёрнут высотой 280px). То есть на планшете одновременно видны и файл, и чат.
- **Расхождение:** Принципиально разная модель центра. Макет: ИЛИ файл ИЛИ чат во всю ширину. Реализация: файл + свёрнутый/развёрнутый док чата снизу — конструкция, которой в планшетном макете нет.
- **Рекомендация:** Либо привести планшет к модели макета (центр = только файл, чат скрыт, возврат кнопкой в шапке файла), либо осознанно зафиксировать док чата как продуктовое улучшение и обновить макет. Сейчас это расхождение задумки и реализации №1.

### [MC2] Ширина левого рельса: макет фиксирует 288px, реализация делает его перетаскиваемым
- **Severity:** P2
- **Экран:** Рабочее пространство (чат и файлы), `da-ws-chat-tablet.png`, `da-ws-files-tablet.png`
- **Задумано (макет):** Рельс — `flex:none; width:288px`, постоянная фиксированная ширина.
- **Реализовано:** `sidebarWidth` — состояние с дефолтом 300px, диапазон 220–520px, перетаскивается мышью (`handleSidebarSplitterMouseDown`) и сохраняется в `localStorage` (`cc_sidebar_width`). На планшете рельс остаётся ресайзящимся, а не фиксированным 288.
- **Расхождение:** И дефолтная ширина (300 vs 288), и сама возможность менять ширину рельса перетаскиванием — отступление от планшетной задумки фиксированного рельса.
- **Рекомендация:** Для планшета задать фиксированную ширину 288px и убрать вертикальный сплиттер-ручку рельса (см. MC3). На десктопе ресайз можно оставить.

### [MC3] Вертикальный сплиттер-ручка между рельсом и центром — отсутствует в макете
- **Severity:** P2
- **Экран:** Рабочее пространство, граница рельс/центр
- **Задумано (макет):** Между рельсом и центром — просто `border-right:1px solid #DDD4C4`, без интерактивной ручки.
- **Реализовано:** Планшет идёт через тот же `return` (после мобильной ветки), где между `Sidebar` и контентом всегда рендерится сплиттер шириной 5px с видимой ручкой-«пилюлей» (строки 327–334) и курсором `col-resize`.
- **Расхождение:** На планшете присутствует визуальный элемент (полоса 5px + ручка 1×32px), которого в макете нет; граница рельса в макете — тонкая линия 1px.
- **Рекомендация:** На планшете (`isTablet`) не рендерить ресайз-сплиттер рельса, оставить только бордер 1px как в макете.

### [MC4] Горизонтальный сплиттер дока чата на планшете — отсутствует в макете
- **Severity:** P2
- **Экран:** Рабочее пространство, открыт файл, планшет
- **Задумано (макет):** В режиме файла нет ни дока чата, ни горизонтального сплиттера (см. MC1).
- **Реализовано:** Когда `chatDockExpanded`, между `FileViewer` и доком чата рендерится горизонтальная ручка-сплиттер `row-resize` высотой 5px (строки 375–388).
- **Расхождение:** Дополнительный интерактивный разделитель, прямое следствие добавленного дока чата (MC1); в задумке планшета его нет.
- **Рекомендация:** Снимается вместе с MC1, если убрать док чата. Иначе — отметить как часть продуктового решения.

### [MC5] Шапка просмотра файла: на планшете нет кнопки «назад к чату», которая есть в макете
- **Severity:** P2
- **Экран:** Рабочее пространство, шапка открытого файла, планшет
- **Задумано (макет):** Шапка файла (`centerFile`) начинается с кнопки `backToChat` — «‹ {sessionTitle}» (chip `#EDE7DA`), это основной способ вернуться от файла к чату, раз центр взаимоисключающий.
- **Реализовано:** На планшете `FileViewer` получает `isFullscreen`, но `onToggleFullscreen={isTablet ? undefined : ...}` (строка 372) — то есть кнопка выхода из fullscreen на планшете отключена; возврат предполагается через постоянно видимый док чата снизу. Отдельной кнопки «назад к чату с названием сессии» в шапке файла, как в макете, нет.
- **Расхождение:** Разные навигационные модели возврата от файла к чату. Макет — явная кнопка в шапке; реализация — переключение внимания между файлом и доком чата.
- **Рекомендация:** Если переходить на модель макета (MC1), добавить в шапку `FileViewer` кнопку «‹ {название сессии}» для возврата к чату.

### [MC6] Карточка «Сессия запущена»: нет индикаторов MCP/пути и подзаголовка «обновлено только что»
- **Severity:** P3
- **Экран:** Рабочее пространство, чат (`da-ws-chat-tablet.png`)
- **Задумано (макет):** Карточка старта сессии показывает `claude-sonnet-4-6 · acceptEdits · ~/dev/acme-api` и справа индикаторы «● 2 MCP», «● 1». Шапка чата центра: подзаголовок `{projectName} · обновлено только что`.
- **Реализовано:** Карточка «Чат запущен» показывает только модель и режим (`claude-opus-4-8[1m] · auto`), без пути и без MCP-индикаторов. Подзаголовка «обновлено только что» / MCP в шапке нет (grep по `ChatPanel.tsx` — совпадений нет).
- **Расхождение:** Отсутствуют MCP-индикаторы, путь рабочей директории в карточке и метка времени в шапке. Не специфично для планшета (общее с десктопом), но видно и на планшетном экране.
- **Рекомендация:** Низкий приоритет. При наличии данных по MCP/времени — добавить; иначе оставить, отметив как сознательное упрощение демо.

### [MC7] Иконка логотипа рельса: «звезда/искра» вместо креста-звёздочки из макета
- **Severity:** P3
- **Экран:** Рабочее пространство, верх рельса (`da-ws-chat-tablet.png`, `da-ws-files-tablet.png`)
- **Задумано (макет):** В плашке логотипа рельса — иконка-крест/звёздочка (`path d="M12 3v18M3 12h18M5.6 5.6..."`) на акцентном квадрате 28×28.
- **Реализовано:** Код рельса (строки 212–216) рисует ровно ту же SVG-иконку, НО на скриншотах `da-ws-*-tablet.png` в шапке отображается другой глиф (восьмилучевая «искра»/звезда). Вероятно, рендерится иной компонент/состояние, чем в просмотренной ветке `!isMobile`.
- **Расхождение:** Визуально логотип на планшетных скриншотах не совпадает с крестом-звёздочкой макета. Требует проверки, какой компонент реально отрисован.
- **Рекомендация:** Сверить фактический рендер логотипа в рабочем пространстве с макетом; привести к единому глифу (крест-звёздочка `#D97757`).

**Счёт по severity:** P1 — 1 (MC1); P2 — 4 (MC2, MC3, MC4, MC5); P3 — 2 (MC6, MC7).

**Топ-3:**
